## v1.2.0 - 2026-07-04

**Full framework coverage.** The `/rgxvisual` suite grows from 4 tabs to 9, covering every user-facing RGX-Framework module:

- **Tooltip** — `Tip:Attach` (builder-at-hover-time), manual `Tip:Show`/`Tip:Hide` with `ANCHOR_TOP`, and a one-shot `HookNative("item")` registration that injects a line into native item tooltips (session-persistent by design — the framework keeps one Blizzard registration per type and pcall-guards every callback).
- **Auras** — `IterateAuras("player", "HELPFUL")` scan with name listing, and a live `WatchUnit` + `OnApplied`/`OnRemoved` chat log whose unsubscribe closures are exercised by the stop button.
- **Minimap** — `MM:Create` with icon, composed tooltip, left-click handler, circular drag with the angle persisted to `RGXVisualTestDB`, plus `Toggle`/`IsShown`/`GetAngle`.
- **Design** — `RGX:Font` one-call styling (defaults and explicit name/size/flags), `RGXDesign:CreateSectionHeader`/`CreateDivider`/`CreateButton` with theme-token hover styling.
- **System** — `RGX:After(2)` one-shot, `RGX:Every(1)` ticker with a live counter, and `RGX:CancelTimer` stop.

Sound is intentionally not tested here — the sound module is a per-addon registry (`Sound:Register` returning a handle) that BLU exercises in production, which is a more honest test than a synthetic registration.

Every API call in the new tabs was verified against RGX-Framework source (`modules/tooltip/tooltip.lua`, `modules/auras/auras.lua`, `modules/minimap/minimap.lua`, `modules/design/design.lua`, `core/systems/runtime.lua`) before being written.

Also: `media/README.md` no longer leaks into the packaged zip.
